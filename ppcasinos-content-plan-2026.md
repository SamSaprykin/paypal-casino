### Technical Implementation

**1. hreflang tags — Essential**

Add to every page that has a language equivalent. Without this, Google may treat translations as duplicate content.

```html
<link rel="alternate" hreflang="en" href="https://ppcasinos.co/suomi-guide/" />
<link rel="alternate" hreflang="fi" href="https://ppcasinos.co/fi/suomi-opas/" />
<link rel="alternate" hreflang="sv" href="https://ppcasinos.co/sv/sverige-guide/" />
<link rel="alternate" hreflang="de" href="https://ppcasinos.co/de/deutschland/" />
<link rel="alternate" hreflang="x-default" href="https://ppcasinos.co/" />
```

**2. URL structure — Use subdirectories, not subdomains**

| ✅ Do this | ❌ Not this |
|---|---|
| `ppcasinos.co/fi/suomi-opas/` | `fi.ppcasinos.co/suomi-opas/` |
| `ppcasinos.co/de/deutschland/` | `de.ppcasinos.co/deutschland/` |

Subdirectories share your domain authority. Subdomains dilute it.

**3. Separate URLs per language — always**

Do not use JavaScript to swap content on the same URL. Google needs separate crawlable URLs for each language.

**4. Canonical tags**

Each language version should self-canonicalize. The English version must NOT be set as canonical for the Finnish/Swedish/German versions.

```html
<!-- On the Finnish page -->
<link rel="canonical" href="https://ppcasinos.co/fi/suomi-opas/" />

<!-- On the English page -->
<link rel="canonical" href="https://ppcasinos.co/suomi-guide/" />
```

**5. Geo-detection — soft suggest only**

Use a banner ("Näytä suomeksi?" for Finnish visitors) rather than auto-redirect. Auto-redirects hurt UX and can confuse crawlers.
